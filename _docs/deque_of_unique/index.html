Defined in header <deque_of_unique>
template<

    class T,
    class Hash = std::hash<T>,
    class KeyEqual = std::equal_to<T>

> class deque_of_unique;

Combining the characteristics of a double-ended queue (std::deque) and an std::unordered_set, the deque_of_unique ensures that all elements are unique and can be accessed or modified from both ends. This makes it ideal for applications where maintaining order and preventing duplicates is essential.

Template parameters

T - The type of the elements.
The requirements that are imposed on the elements depend on the actual operations performed on the container. Generally, it is required that element type is a complete type and meets the requirements of Erasable, but many member functions impose stricter requirements. (since C++11)
Hash - A unary function object type that hashes a Key. The type must meet the requirements of Hash. Defaults to std::hash<T>.
KeyEqual - A binary predicate that takes two arguments of the same type as the elements and returns a bool. The type must meet the requirements of BinaryPredicate. Defaults to std::equal_to<T>.

Example 

#include <iostream>
#include "deque_of_unique.h"
 
int main()
{
    // Create a deque_of_unique containing integers
    containerofunique::deque_of_unique<int> d = {3, 5, 20, 9};
     
    // Duplicate is silently ignored
    d.push_front(3);
    d.push_back(28);
     
    for (int n : d)
        std::cout << n << ' ';
    std::cout << '\n';
}

Output:

3 5 20 9 28
